🚕 What's the story?

This is a coding competition. Each participant writes their own Matching algorithm — that is, a function that, in a live Uber simulation, decides which driver is assigned to which rider.

The game engine simulates a real world: drivers move, riders make requests, and fares, ratings, and driver fatigue are all computed. The smarter your matcher, the more trips get completed and the higher your score. You can watch each world's live map and scoreboard at the top of this page.

📋 Rules & scoring

🏆 Your score is your total revenue — the sum of fares from every completed trip. The Results board ranks matchers by revenue, highest first. Complete more (and longer) trips and avoid cancellations.
💵 Trip farefare = 5 + 1.5 × trip distance — added to your revenue on each completed trip (distance from origin to destination, not the pickup drive).
❌ CancellationA rider waits at most 5 minutes from request to pickup. Miss that window — or never assign a driver — and the trip cancels for zero revenue.
⭐ Rider ratingStars from the wait between request → pickup. Match fast to score high.
⭐ Driver ratingStars from the time between assignment → pickup. Pick the nearest free driver.
😴 DriversAccept every trip, one at a time. After 30 min with no trip they go offline (sleep), and wake up 60 min later.

Both ratings use the same scale (minutes of wait):

⏱️ Ticks & timing

The world advances in discrete steps called cycles (ticks). Each cycle your matcher is called once: the engine sends you the current world state, your decide() returns assignments, then the engine moves the drivers, settles completions and cancellations, and updates the scores.

⏱️ One cycle= 1 second of real time on this arena, and represents 1 game-minute. Every rule time is in game-minutes — so the 5-minute pickup cap is 5 cycles, and the 30-minute sleep is 30 cycles.
🏁 Session length240 cycles (≈4 real minutes, = 4 game-hours). When the last cycle runs, the world finishes and is saved to the Results board.
🧍 Each cycleAbout 10 new ride requests appear (on average), in a world of 100 drivers.
⚡ Your turnSend your assignments for the current tick before the next cycle. Assignments tagged with an outdated tick are rejected.

(These are the engine's current settings and can be tuned server-side.)

🎮 How do I play?

1Get the sample code
Clone or download the repository from GitHub.
2Pick a language
A ready-to-run sample client is included for both Node.js and Python.
3Write your algorithm
Change only the decide() function and run it — your world shows up on this page.

First, get the code:

git clone https://github.com/Real-World-Games/UberSimulation
cd UberSimulation

Then run one of the sample clients (each connects to this arena):

🟢 Node.js (Node 22+)
cd client/node-client
npm install
MATCHER_NAME="Team Alpha" npm start
🐍 Python (Python 3.10+)
cd client/python-client
pip install -r requirements.txt
MATCHER_NAME="Team Alpha" python sample_client.py

On start, the client prints a link to your live world — and your map appears in the arena at the top of this page. Then open the sample client and change the decide() function to write your own matching strategy.
