🎬 Your runs & replays

🚕 What's the story?

This is a coding competition. Each participant writes their own Matching algorithm — that is, a function that, in a live Uber simulation, decides which driver is assigned to which rider.

The game engine simulates a real world: drivers move, riders make requests, and fares, ratings, and driver fatigue are all computed. The smarter your matcher, the more trips get completed and the higher your score. Runs finish in moments, then land on the leaderboard above — and you can watch a full replay of any run.

📋 Rules & scoring

🏆 Your score is your total revenue — the sum of fares from every completed trip. The Results board ranks matchers by revenue, highest first. Complete more (and longer) trips and avoid cancellations.
💵 Trip farefare = 5 + 1.5 × trip distance — added to your revenue on each completed trip (distance from origin to destination, not the pickup drive).
❌ CancellationA rider waits at most 5 minutes from request to pickup. Miss that window — or never assign a driver — and the trip cancels for zero revenue.
⭐ Rider ratingStars from the wait between request → pickup. Match fast to score high.
⭐ Driver ratingStars from the time between assignment → pickup. Pick the nearest free driver.
😴 DriversAccept every trip, one at a time. After 30 min with no trip they go offline (sleep), and wake up 60 min later.

⏱️ Ticks & timing

The world advances in discrete steps called cycles (ticks). Each cycle your matcher is called once: the engine sends you the current world state, your decide() returns assignments, then the engine moves the drivers, settles completions and cancellations, and updates the scores.

⏱️ One cyclerepresents 1 game-minute. Every rule time is in game-minutes — so the 5-minute pickup cap is 5 cycles, and the 30-minute sleep is 30 cycles.
⚡ Full speedThe arena runs as fast as your matcher answers — the world advances the instant you submit, not on a fixed clock. Answer faster, finish sooner. (A slow/dead client is auto-advanced after a timeout so it can't stall.)
🏁 Run lengthEach city runs 240 cycles (= 4 game-hours); your matcher plays every city in turn on one connection. When the last city's last cycle runs, the run finishes and its combined total is saved to the Results board.
🧍 Each cycleNew ride requests appear (Poisson). How many — and how many drivers exist — varies by city; see “The cities” below.
⚡ Your turnSend your assignments for the current tick; the engine immediately advances. Assignments tagged with an outdated tick are rejected.

(These are the engine's current settings and can be tuned server-side.)

🎮 How do I play?

1Create an account
Register up top with a username + password. You get a personal API token — your client uses it to prove which player it is.
2Get the sample code
Clone the repo from GitHub. A ready-to-run client is included for both Node.js and Python.
3Write your algorithm
Change only the decide() function, run with your token — your run lands on the leaderboard above, and you can watch its replay.
🔑 Log in above to get your API token, then pass it to the client as the TOKEN environment variable. Every session is tied to your account, so the leaderboard is per player and your submissions are public for everyone to see.

First, get the code:

git clone https://github.com/Real-World-Games/UberSimulation
cd UberSimulation

Then run one of the sample clients with your token (each connects to this arena):

🟢 Node.js (Node 22+)
cd client/node-client
npm install
TOKEN=your_api_token npm start
🐍 Python (Python 3.10+)
cd client/python-client
pip install -r requirements.txt
TOKEN=your_api_token python sample_client.py

The run finishes in moments and lands on the leaderboard at the top of this page, where you can watch its replay. Then open the sample client and change the decide() function to write your own matching strategy.
